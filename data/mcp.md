<h1 id="model-context-protocol-mcp">Model Context Protocol (MCP)</h1>
<p>PydanticAI supports <a href="https://modelcontextprotocol.io">Model Context Protocol (MCP)</a> in three ways:</p>
<ol>
<li><a href="../agents/">Agents</a> act as an MCP Client, connecting to MCP servers to use their tools, <a href="client/">learn more …</a></li>
<li>Agents can be used within MCP servers, <a href="server/">learn more …</a></li>
<li>As part of PydanticAI, we're building a number of MCP servers, <a href="#mcp-servers">see below</a></li>
</ol>
<h2 id="what-is-mcp">What is MCP?</h2>
<p>The Model Context Protocol is a standardized protocol that allow AI applications (including programmatic agents like PydanticAI, coding agents like <a href="https://www.cursor.com/">cursor</a>, and desktop applications like <a href="https://claude.ai/download">Claude Desktop</a>) to connect to external tools and services using a common interface.</p>
<p>As with other protocols, the dream of MCP is that a wide range of applications can speak to each other without the need for specific integrations.</p>
<p>There is a great list of MCP servers at <a href="https://github.com/modelcontextprotocol/servers">github.com/modelcontextprotocol/servers</a>.</p>
<p>Some examples of what this means:</p>
<ul>
<li>PydanticAI could use a web search service implemented as an MCP server to implement a deep research agent</li>
<li>Cursor could connect to the <a href="https://github.com/pydantic/logfire-mcp">Pydantic Logfire</a> MCP server to search logs, traces and metrics to gain context while fixing a bug</li>
<li>PydanticAI, or any other MCP client could connect to our <a href="run-python/">Run Python</a> MCP server to run arbitrary Python code in a sandboxed environment</li>
</ul>
<h2 id="mcp-servers">MCP Servers</h2>
<p>To add functionality to PydanticAI while making it as widely usable as possible, we're implementing some functionality as MCP servers.</p>
<p>So far, we've only implemented one MCP server as part of PydanticAI:</p>
<ul>
<li><a href="run-python/">Run Python</a>: A sandboxed Python interpreter that can run arbitrary code, with a focus on security and safety.</li>
</ul>
